# Persona Best Practices

## What Makes a Good Persona?

A good persona is not a demographic profile. It's a behaviorally-grounded archetype that drives design decisions.

## Core Principles

### 1. Behavior-First, Not Demographics

Bad persona:
> "Sarah, 32, female, lives in Shanghai, income ¥15K/month"

Good persona:
> "Sarah, efficiency-driven shopper: searches directly, compares 3+ options, completes purchase within 10 minutes"

### 2. Primary Persona Rule

- **Maximum 2 primary personas** per product
- If you have more than 2, you're trying to serve everyone
- Secondary personas handle edge cases and special scenarios

### 3. Evidence-Based

Every persona element should be traceable to research:

| Persona Element | Evidence Source |
|-----------------|-----------------|
| Goals | User interviews, task analysis |
| Behaviors | Analytics, log data, observation |
| Pain points | Support tickets, usability tests |
| Quote | Direct user quote from interview |

### 4. Actionable Design Guidance

Each persona should answer:
- What does this persona need to accomplish?
- What frustrates them most?
- What design decisions does this inform?

## Persona Validation Checklist

- [ ] Based on real user research (not assumptions)
- [ ] Has a clear primary/secondary designation
- [ ] Includes behavioral data, not just demographics
- [ ] Has a direct user quote
- [ ] Lists 3-5 specific goals
- [ ] Identifies 3-5 specific pain points
- [ ] Includes design implications
- [ ] Reviewed with the team
- [ ] Updated within the last 12 months

## When to Update Personas

| Trigger | Action |
|---------|--------|
| New user research | Validate existing personas |
| Major product change | Check if personas still apply |
| Market expansion | Add new personas if needed |
| 12+ months since last review | Full persona refresh |

## Connecting to the Ecosystem

- **UDM** → Use UDM interview methods to collect persona research data
- **JTBD** → Cluster users by "jobs" to discover natural persona segments
- **QuantUX** → Validate persona behavior patterns with analytics data
- **VPD** → Map persona goals and pains to the Value Proposition Canvas
- **SWD** → Present persona stories to stakeholders with compelling data narratives
